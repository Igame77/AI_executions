# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

Дополнительные параметры для настройки отчета:

- `--max-hist-columns` – максимальное количество числовых колонок для гистограмм (по умолчанию 6)
- `--top-k-categories` – количество top-значений для категориальных признаков (по умолчанию 3)
- `--title` – заголовок отчёта в Markdown (по умолчанию "EDA-отчет")
- `--min-missing-share` – порог доли пропусков для выявления проблемных колонок (по умолчанию 0.02)

Пример с использованием всех параметров:

```bash
uv run eda-cli report data/example.csv \
  --out-dir "detailed_report" \
  --max-hist-columns 10 \
  --top-k-categories 5 \
  --title "Детальный анализ данных" \
  --min-missing-share 0.05 \
  --sep ";" \
  --encoding "windows-1251"
```

В результате в указанном каталоге появятся:

- `report.md` – основной отчёт в Markdown со всеми разделами;
- `summary.csv` – таблица по колонкам с подробной статистикой;
- `missing.csv` – статистика пропусков по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/` – папка с top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

## Структура отчёта

Markdown-отчёт включает следующие разделы:

1. **Заголовок** (настраивается через `--title`)
2. **Качество данных (эвристики)** – автоматическая оценка качества с флагами
3. **Колонки** – ссылка на подробную таблицу
4. **Пропуски** – статистика и визуализация пропусков
5. **Корреляция числовых признаков** – матрица корреляций
6. **Категориальные признаки** – top-k значений для каждой категориальной колонки
7. **Проблемные признаки** – колонки с долей пропусков выше порога (настраивается через `--min-missing-share`)
8. **Гистограммы** – визуализация распределения числовых данных

## Новые возможности

### Настройка визуализаций
- `--max-hist-columns` позволяет ограничить количество гистограмм для больших датасетов
- `--top-k-categories` настраивает детальность анализа категориальных признаков

### Анализ качества
- `--min-missing-share` задает порог для выявления проблемных колонок с пропусками
- Автоматическая оценка качества данных с эвристиками

### Кастомизация
- `--title` позволяет задать заголовок отчёта
- Все параметры имеют разумные значения по умолчанию
